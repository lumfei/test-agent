# API Test Agent 启动教程 🧪

> **给定一个 OpenAPI 文档 URL，Agent 自动完成解析→分析→生成用例→执行→验证报告全流程。**
> 基于 LangGraph + DeepSeek，5 节点 Pipeline，SSE 实时推送，3 种报告格式（HTML/Markdown/JSON）。

---

## 准备清单（5 分钟）

- [ ] Python 3.10+：`python --version`
- [ ] 安装依赖：`pip install -r requirements.txt`
- [ ] 配置 API Key：`cp .env.example .env` → 编辑填入 `DEEPSEEK_API_KEY`
- [ ] 目标 API 就绪（本地 `http://localhost:8000` 或任意 OpenAPI URL）

> **不需要 Docker**（Qdrant 非必需，仅用于高级 RAG 功能）：`python setup_and_test.py --skip-docker` 可直接跑。

## 快速启动

### 方式 1：一键跑通（推荐面试用）

```bash
# 直接用 Python 跑一次完整 E2E
cd api-test-agent
python -c "
import asyncio
from src.agent.graph import run_agent
state = asyncio.run(run_agent('http://localhost:8000/openapi.json'))
print(f\"Pass: {sum(1 for r in state['execution_results'] if r['passed'])}/{len(state['execution_results'])}\")
print(f\"Report: {state['report_path']}\")
"
```

运行完后 `reports/` 目录下生成 3 份报告：
- 🌐 `report_xxx.html` — 富交互页面（仪表盘 + 搜索 + 图表）
- 📝 `report_xxx.md` — 纯文本
- 📊 `report_xxx.json` — 结构化数据

### 方式 2：Streamlit Dashboard

```bash
streamlit run frontend/app.py
# 打开 http://localhost:8501
```

Dashboard 支持两种模式：

| 模式 | 说明 | 需要后端？ |
|------|------|-----------|
| 🌐 在线模式 | 连接 FastAPI 后端，启动测试 + 实时进度 + 历史 | 需要 |
| 📁 离线模式 | 直接读 `reports/*.json`，图表 + 筛选 + 下载 | **不需要** |

离线模式下直接加载已有的 JSON 报告，Pie 图/条形图/筛选/失败详情全部可用。

### 方式 3：FastAPI 后端（完整功能）

```bash
# 终端 1：启动后端
uvicorn src.main:app --reload --port 8002

# 终端 2：测试
curl -X POST http://localhost:8002/api/test/run \
  -H "Content-Type: application/json" \
  -d '{"spec_url": "http://localhost:8000/openapi.json"}'

# SSE 实时进度
curl http://localhost:8002/api/test/stream/{run_id}

# 下载报告
curl http://localhost:8002/api/test/report/{run_id}?format=html > report.html
```

## Agent Pipeline 流程

```
Spec URL
   │
   ▼
[1. Parse]          ← 抓取并解析 OpenAPI 文档
   │
   ▼
[2. Analyze]        ← LLM 分析端点风险等级，确定测试优先级
   │
   ▼
[3. Generate]       ← LLM 生成测试用例（normal/boundary/error/security）
   │
   ▼
[4. Execute]        ← 真实 HTTP 请求发送，记录响应
   │
   ▼
[5. Validate]       ← Schema 验证 + LLM-as-Judge 语义分析 → 生成报告
```

4 类自动生成的用例：
- **normal**: 合法参数，期望 2xx
- **boundary**: 空 body / 超大 body / unicode / 极端值
- **error**: 错误 HTTP 方法 / 错误 Content-Type / 畸形 JSON
- **security**: SQL 注入 / XSS / 超长字符串 / 路径遍历

| 层级 | 技术 | 面试要说 |
|------|------|----------|
| 编排 | LangGraph StateGraph | "5 节点状态图，带 checkpoint，支持断点续跑和时间旅行" |

| 日志 | loguru | "彩色终端 + 文件滚动归档，结构化日志" |
| 向量存储 | Qdrant | "OpenAPI spec 向量化存储，语义检索历史用例" |
| 可观测 | LangFuse (可选) | "LLM 调用链追踪 + 成本统计" |
| 工程化 | pyproject.toml + pytest-cov | "现代 Python 项目标准，覆盖率 HTML 报告" |

## 常见问题

**Q: DeepSeek API 连不上 / SSL 错误？**
> 系统代理干扰。已修复——LLM 客户端使用 `trust_env=False` 绕过代理直连。

**Q: 测试通过率低？**
> 正常。目标 API 的端点需要特定参数或认证才能 2xx，Agent 会标记为 boundary/error 类测试。

**Q: 不想装 Docker / Qdrant？**
> 不影响核心功能。Qdrant 仅用于高级 RAG 检索，跳过 Docker 检查：`python setup_and_test.py --skip-docker`

**Q: 覆盖率只有 25%？**
> 正常。单元测试仅覆盖纯逻辑 `src/tools/` 模块，其余模块（agent 节点、LLM、存储）依赖外部服务，由 E2E 测试覆盖。

**Q: 报告中文乱码？**
> 使用 UTF-8 终端（Windows Terminal / Git Bash），不要用 cmd。HTML 报告强制 UTF-8 编码，不受终端影响。
