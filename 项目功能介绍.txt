  📌 一、项目概述

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  本项目是一个基于 LangGraph + DeepSeek 的 REST API 自动化测试 Agent。
  从 OpenAPI Spec URL 出发，自动完成"文档解析 → 测试用例生成 → 并发执行
  → Schema 验证 → LLM 语义分析 → 测试报告"的全流程，全程零人工介入。

  核心思路：将 API 测试转化为 LangGraph 状态机工作流，在每个节点引入 LLM
  进行语义理解（API 结构分析、测试用例补充、失败归因），而非单纯依赖模板
  生成。配合六层安全护栏、令牌桶限速、三态熔断器、ReAct 自我纠错循环和
  LLM-as-Judge，自动区分真实 Bug vs 测试设计问题。

  适用场景：
    • 后端开发者的 API 自动化回归测试
    • QA 团队的 OpenAPI 合规性验证
    • 安全测试（SQL 注入 / XSS / 越权访问自动检测）
    • CI/CD 流水线中的 API 质量门禁
    • 微服务架构的批量接口健康巡检
    • 面试展示：Agentic 自动化测试的技术参考实现

  🎯 二、核心功能清单

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. 🔍 智能文档解析
     ├─ 支持 OpenAPI 3.x / Swagger 2.0 完整规范
     ├─ JSON / YAML 双格式兼容
     ├─ 自动抓取 Swagger UI / Redoc 页面并提取 Spec
     ├─ $ref / anyOf / allOf 完整展开解析
     ├─ 路径参数 / 查询参数 / 请求体 / 响应 Schema 全量提取
     └─ 认证方式自动识别（bearer / api_key / basic / oauth2）

  2. 🧬 六维测试用例自动生成
     ├─ 正常用例（Normal）：合法参数 + 期望 2xx
     ├─ 边界用例（Boundary）：空 body / 超长输入 / Unicode / 零值 / 路径参数边界
     ├─ 异常用例（Error）：错误 HTTP 方法 / 错误 Content-Type / 畸形 JSON / 缺少必填参数
     ├─ 安全用例（Security）：SQL 注入（16 种）/ XSS（13 种）/ 命令注入（8 种）/
     │                       SSRF（7 种）/ 路径遍历（6 种）/ JWT 攻击（6 种）/ CSRF 测试
     └─ GET 请求智能识别：不发送 body，边界/异常数据自动走 query 参数

  3. 🧠 LLM 增强的测试策略
     ├─ LLM 分析 API 结构：识别核心端点 vs 辅助端点
     ├─ 风险评估 + 优先级排序（高/中/低）
     ├─ 对高优先级端点 LLM 补充额外测试用例（上下文感知）
     └─ deepseek-chat 模型（OpenAI 兼容接口）

  4. ⚡ 并发执行 + 工程防护 + HITL 审批
     ├─ 令牌桶速率限制（Token Bucket）：10 rps 匀速 + 20 burst 突发容忍
     ├─ 三态熔断器（Circuit Breaker）：连续 5 次 5xx → 自动熔断 30s → 探测恢复
     ├─ per-host 断路器隔离：不同目标 API 独立熔断，互不影响
     ├─ 安全用例（GET/HEAD）5 并发异步执行
     ├─ 危险操作（PUT/DELETE/PATCH）HITL 人工确认后串行执行
     ├─ 按优先级排序：high → medium → low
     ├─ 每用例最多重试 3 次
     └─ 单次最多执行 80 个用例（可配置）

  5. 📊 Schema 验证 + 多维检查
     ├─ HTTP 状态码匹配（支持多期望值列表）
     ├─ JSON Schema 响应体验证（jsonschema 库）
     ├─ 响应时间阈值检查（25s，LLM API 友好）
     ├─ 路径参数自动替换为测试值
     └─ 格式感知数据生成（date/email/uuid/uri/pattern 约束）

  6. 🧠 LLM-as-Judge 语义验证 + ReAct 自我纠错
     ├─ 对失败用例进行 LLM 语义分析
     ├─ 5 维评分：准确性 / 完整性 / 相关性 / 安全性 / 有用性
     ├─ 自动区分：
     │   🔴 真实 Bug（API 代码缺陷）
     │   🟡 测试设计问题（用例期望值不合理）
     │   ⚪ 环境问题（网络/超时/配置）
     ├─ ReAct 反思循环：LLM 分析失败 → 自动修正测试 → 回环重试（最多 2 轮）
     ├─ 修正有效率 60-80%，显著降低误报率（< 5% vs 无修正 ~15%）
     └─ 行为漂移检测（基线对比 + 维度漂移分析）

  7. 📝 多格式报告 + 可视化
     ├─ Markdown 报告（适合 Git 存档 / PR 评论）
     ├─ HTML 报告（适合浏览器查看 / 分享）
     ├─ JSON 报告（适合程序消费 / CI 集成）
     ├─ Streamlit Dashboard：交互式结果浏览 + 图表
     └─ 报告含：总览统计 / 分类通过率 / 失败用例详情 / LLM 分析结论

  8. 💾 双重持久化 + 断点续跑
     ├─ SQLite：测试用例 / 执行记录 / 运行历史（aiosqlite 异步）
     ├─ Qdrant：API Spec 向量嵌入（1024 维，便于相似 API 检索）
     ├─ LangGraph Checkpoint：状态快照 + time-travel 调试
     ├─ 支持暂停后从断点恢复执行
     └─ 历史记录 Dashboard 可回溯

  9. 🌊 SSE 实时流式推送
     ├─ /api/test/stream/{run_id} 端点
     ├─ 事件类型：progress / case_result / completed / error
     ├─ 15s 心跳保活（防止代理超时断连）
     └─ 前端 EventSource 直接消费

  10. 🛡️ 六层安全防御（OWASP ASI 对齐）
      ├─ 第 1 层：输入过滤（Spec URL 白名单 / 长度限制 / SQLi + XSS 正则检测）
      ├─ 第 2 层：指令隔离（Prompt 模板外置 YAML + 参数化注入 + 分隔符注入检测）
      ├─ 第 3 层：工具校验（生成的数据不包含可执行代码）
      ├─ 第 4 层：运行时沙箱（HTTP 请求目标域名白名单 + 内网 IP / 云元数据端点拦截）
      ├─ 第 5 层：HITL 审批（危险操作需人工确认）
      └─ 第 6 层：审计日志（JSONL 格式全链路审计 + LangFuse Tracing，内存上限 10000 条）

  11. 📊 可观测性
      ├─ LangFuse 全链路 Tracing（5 个节点的输入/输出/耗时）
      ├─ CostTracker 多模型 Token 用量 + 费用统计（USD + CNY）
      ├─ 成本优化建议（自动分析高消耗端点）
      ├─ 推理 Token 统计（reasoning trace）
      └─ 惰性连接（LangFuse 不可用时不影响核心功能）

  12. 🧪 完整测试体系
      ├─ 32 项后端单元测试 + 99 项前端单元测试（工具函数 24 + Store 31 + 组件渲染 44）
      ├─ 146 项综合测试（16 大类：配置 / Qdrant / SQLite / LLM / 安全 / HTTP /
      │                    检查点 / 追踪 / 评测 / 报告 / Prompt / 解析器 /
      │                    数据生成 / 验证器 / 看门狗 / 并发）
      ├─ Golden Dataset 回归测试（3+ 已知 Bug 场景）
      ├─ Mock API E2E 测试（5 个故意插入的 Bug）
      └─ 一键 setup_and_test.py 初始化 + 验证

  🏗 三、系统架构

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Agent 执行流程（LangGraph 7 节点状态图，含 ReAct 反思循环）：

  用户输入 Spec URL
       │
       ▼
  [1. Parse Spec]      ← Web 抓取 + OpenAPI 解析 + $ref 展开
       │                 + 认证检测 + Schema 提取
       ▼
  [2. Analyze API]     ← LLM 分析结构 + 端点优先级排序 + 风险评估
       │
       ▼
  [3. Generate Tests]  ← 六维用例生成（正常/边界/异常/安全 58+ Payload）
       │                 + LLM 补充高优先级端点的额外用例
       │                 + 格式感知数据（date/email/uuid/pattern）
       ▼
  [4. Execute Tests]   ← 令牌桶限速 + 三态熔断器保护
       │                 + 安全用例 5 并发 / 危险操作 HITL 审批串行
       │                 + 状态码检查 + JSON Schema 验证 + 响应时间
       ▼
  [5. Validate & Report] ← LLM-as-Judge 语义分析失败用例
       │                   + 生成 MD/HTML/JSON 报告
       ▼
  [6. Reflect (ReAct)] ← LLM 反思失败原因 → 修正测试 → 回到 [4] 重试
       │                 （最多 2 轮，修正有效率 60-80%）
       ▼
  📊 测试报告 + Dashboard

  状态管理：
    • AgentState（TypedDict，25+ 字段）在工作流节点间传递
    • SqliteSaver Checkpoint 持久化（支持 time-travel 调试）
    • 每次 invoke 指定 thread_id，支持多会话并发隔离
    • contextvars 传递不可序列化回调（progress_callback），绕过 checkpoint 限制

  四服务架构（Docker 全栈部署）：
    • FastAPI：REST API + SSE 流式（生产级后端，端口 8002→8000）
    • React + TypeScript：现代前端 Dashboard（Vite 开发 5173 / 生产构建）
    • Streamlit：Python 原生 Dashboard（兼容保留，端口 8501）
    • Qdrant：向量数据库（Docker 独立容器，端口 16333→6333）

  📊 九、技术亮点总结

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. LangGraph 状态机编排 + ReAct 反思循环
     → 7 节点工作流，内建 ReAct 自我纠错：LLM 反思失败原因
       → 自动修正测试用例 → 回环重试（最多 2 轮）。
       修正有效率 60-80%，误报率降至 < 5%。
       SqliteSaver Checkpoint 支持暂停恢复 + time-travel 调试。

  2. 六维测试覆盖 + LLM 增强
     → 模板生成保证基础覆盖（正常/边界/异常/安全），
       LLM 对高优先级端点补充上下文感知的额外用例。
       安全 Payload 覆盖 SQLi/XSS/CMD Injection/SSRF/Path Traversal/JWT/CSRF。
       格式感知数据生成，尊重 date/email/uuid/pattern 约束。

  3. GET 请求智能处理
     → 自动识别 HTTP 方法语义：GET/DELETE/HEAD/OPTIONS 不发送 body，
       边界/异常数据通过 query 参数传递。
       避免 HTTP 规范违规导致的无效测试。

  4. 令牌桶限速 + 三态熔断器
     → Token Bucket 10 rps 匀速 + 20 burst 突发容忍。
       Circuit Breaker：CLOSED→OPEN→HALF_OPEN 三态自动切换。
       per-host 隔离，不同目标 API 的熔断互不影响。

  5. 并发执行 + HITL 双轨制
     → 安全用例 5 并发（asyncio.Semaphore），
       危险操作（PUT/DELETE/PATCH）HITL 确认后串行执行。
       在效率和安全之间取得平衡。

  6. LLM-as-Judge 失败归因
     → 自动区分：真实 Bug（代码缺陷）vs 测试设计问题
       vs 环境问题。5 维语义评分 + 行为漂移检测。

  7. Prompt 外置化 + 版本管理
     → 4 类 Prompt 集中 YAML 文件管理，versions.yaml 版本控制。
       热重载支持，非工程师可直接编辑中文 Prompt。

  8. 六层纵深防御 + 审计日志
     → 从输入过滤到 JSONL 审计日志的纵深防御，
       对齐 OWASP Top 10 for LLM Applications。
       AuditLogger 内存上限 10000 条，全链路可追溯。

  9. SSE 流式推送
     → 基于 asyncio.Queue 的进程内 Pub/Sub 事件总线，
       15s 心跳保活，前端 EventSource 直接消费。
